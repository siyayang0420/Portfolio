EasyStock is a Saas website helps high-end furniture company organize inventory effectively and efficiently. The main goal of this project is to apply the agile development process and sprint out the main features and solutions.

Challenge

Create a user-friendly website for daily use - visual-pleasing, easy-to-use, with a clean and simple design that indicates the company's projected image and provides easy navigation throughout the platform. Minimizing the steps, and at the same time achieve the user's maximum goal.

  • Research in the industry and interview multiple times with potential users.
  • Figure out the logistic logic behind the inventory system, to better serve our users.
  • Come out mainly solutions and features that help users systematically organized the inventory.
  • Make accurate changes and improvements based on the feedback.

Problems&Improvements

Our approach was to present the site as a visual editorial platform with useful features based on the company's image and the value. Each solution and feature focus on blending into user's daily routine and aim to become a productive tool for them to use.


Problem

The user usually orders a different product with different variants in multiple amounts, so sending emails one by one to the manufacturer is not ideal and practical.

Solution

Adding an extra session for “Pending orders”. In this session, it will list out all of the orders since users will place multiple orders at one time for each kind of product. And based on the frequent situation, in which the furniture store will prepare a list of products they need to order before they send out the email, the “Pending orders” session will be a whole list of it. This list is for the manager or supervisor to approve before the list is sent out to the manufacturers.





Problem

Initial there is no additional information need to show, but the user wants to see more about the product in terms of its variants.

solutions

Instead of getting into the product detail page, click on the product itself will show a slide down sub-list with the product’s variants and stock availability corresponding with its variants on the same page. This will reduce the deciding process and display it in a cleaner and more concise layout.


Problem

Users would like to know the price for each customized element and the base price of the product.

solutions

Added base price in the product detail page, when adding a new variant option, the option value will show up in motion(animation) to tell the users the option values can be modified.


Problem

The sold quantity for different categories on the category card is not valuable for the user.

Before
After

solutions

The category card will display the sales amount in dollars instead of sold quantity. According to user testing, sales is one of the KPI for their company, so to indicate amount of sold is more accurate.


Problem

The user is confused when trys to see the order of the products.

Before
After

solutions

Added sorting icons to indicate the list can be ordered by users, based on our observation on the tester, these table items are the conceptual models for the tester, where she expects these table items act as sorting buttons at the same time, but she seems confused without seeing something that is indicating the list can be sorted.


Changes and improvements


1. Users would like to see the details of the plan being repetitive on the payment page to remind users of the plan details. Also, users might change their minds and pick other plans, a reused element of the dropdown menu is been used here to offer flexible options to users and to make the design consistent.

2. The symmetrical button layout provides equal weight in visual, which is what we don’t want, to create a stronger call to action button and reduce users’ attention on clicking “cancel”, we decided to enlarge the size of the “pay” button and make “cancel” button less visible.

3. Providing checkboxes to users when selecting orders is convenient. In user testing, the user was expecting only place a few orders at the time since those orders are in a hurry and the rest are not.
